# SmartWaste AI V2 — Comprehensive Project Analysis

This is a complete architectural and operational summary of the SmartWaste AI rural plastic waste management platform, derived directly from the application source code.

## 1. PROJECT OVERVIEW
**SmartWaste AI** is a full-stack, AI-powered municipal waste collection and optimization platform tailored for rural and urban areas (specifically designed around Raipur, Chhattisgarh). It acts as a comprehensive ecosystem to connect citizens, administrative managers, collection workers, self-help groups (SHGs), and recyclers.

*   **Backend**: Python / FastAPI
*   **Database**: PostgreSQL with PostGIS (Spatial queries, GeoAlchemy2)
*   **AI Vision**: Google Gemini 1.5 Flash for image analysis
*   **AI Routing**: Google OR-Tools VRP (Vehicle Routing Problem) Solver
*   **Web Dashboard**: React + Vite + Tailwind CSS
*   **Mobile App**: React Native (Expo)
*   **Infrastructure**: Dockerized (Single command `docker compose up --build` deploy)

## 2. DATABASE MODELS
The platform relies on a relational architecture powered by SQLAlchemy.
*   **Zone**: Operational boundaries (`id`, `name`, `center_lat`, `center_lng`, `radius_km`).
*   **User**: Role-based access accounts (`id`, `email`, `role` [admin, sub_admin, shg, collector], `zone_id`, `is_active`).
*   **Bin**: Physical dumpsters (`id`, `label`, `location` [PostGIS POINT], `status`, `fill_level`, `capacity_kg`, `zone_id`).
*   **BinReport**: Public safety/citizen reports (`id`, `bin_id`, `image_url`, `fill_level`, `waste_type`, `urgency`, `ai_confidence`, `status`, `verified_by`).
*   **Route / RouteStop**: Auto-generated paths for collectors (`id`, `collector_id`, `date`, `total_distance`, `optimized` flag, linked to multiple `RouteStop` checkpoints).
*   **Collection**: Audit logs for physical retrievals (`id`, `collector_id`, `bin_id`, `kg_collected`, `collected_at`).
*   **SHGReport**: Reports specifically generated by SHG members logging plastic weight and fill levels.
*   **Recycler / RecyclerBid**: Entities that purchase collected plastic waste and the transaction logs attached to them.

> **Relationships**: Zones encapsulate Bins and Users (Sub-Admin, SHG, Collector). Bins own Reports and RouteStops. Collections track historical actions by Collectors on Bins. Restrictive foreign key constraints govern all relationships.

## 3. BACKEND API ENDPOINTS
Endpoints are segregated securely by authentication roles across multiple routers.

**Auth Router** (`/api/auth`)
*   `POST /login`: Generate JWT access token
*   `GET /me`: Get current authenticated User profile

**Public Router** (`/api/public` - *Unauthenticated*)
*   `GET /bins`: Fetch map data for all bins
*   `POST /report`: Submit citizen bin issue (Trigger Gemini AI)
*   `GET /report/{id}/status`: Track AI/Verification status
*   `GET /live-status`: Simulated truck tracking data

**Collector Router** (`/api/collector`)
*   `GET /route/today`: Fetch today's optimized stops
*   `POST /bins/{bin_id}/collect`: Mark specific bin picked up (Enforces 50m Geofence)
*   `GET /history`: Collector's past logs
*   `GET /stats`: Monthly/All-time kg collection stats

**SHG Router** (`/api/shg`)
*   `GET /bins`: View bins in assigned zone
*   `POST /bins/{bin_id}/report`: Upload fill level and manual validation
*   `GET /reports` & `/history`: Audit of prior interventions
*   `GET /schedule` & `/stats`: Timetables and worker analytics

**Sub-Admin Router** (`/api/subadmin` - *Scoped to specific Zone*)
*   `GET /reports` & `/pending-reports`: View incoming citizen reports
*   `POST /reports/{id}/verify`: Approve report — **Triggers Route Optimization**
*   `POST /reports/{id}/reject`: Soft-delete/ignore report
*   `GET /dashboard`, `/analytics`, `/bins`, `/routes`: Zone-scoped performance aggregators

**Admin Router** (`/api/admin` - *Platform-Wide*)
*   CRUD operations for `users` and `bins` (`GET`, `POST`, `PUT`, `DELETE`).
*   `GET /dashboard`, `/analytics`: High-level metrics (CO2 reduced, fuel saved).
*   `POST /routes/generate`: Force-run AI VRP routing manually for any/all zones.
*   `GET /export/excel` & `/export/csv`: Export compliance and history metrics.
*   `PUT /settings`: Global app configuration (Geofence radius, spam timeouts).

**Recycler Router** (`/api/recyclers` & `/api/admin/recyclers`)
*   `GET /api/recyclers` (Public): List available recycling centers.
*   `POST /api/recyclers/register` (Public): Form signup for a new recycling company.
*   `POST /api/recyclers/{id}/bid` (Admin/SubAdmin): Sell processed plastic to a recycler.
*   `GET /api/admin/recyclers/stats`: Global plastic market revenue logic.

## 4. WEB DASHBOARD - COMPLETED FEATURES
*   **Admin Dashboard**: Platform oversight, total collected plastic metrics, user/bin management consoles, global analytics (charts/maps), settings configurations, Excel/CSV exporting capabilities. View total recycler earnings.
*   **Sub-Admin Dashboard**: Geared towards report moderation. Incoming citizen reports queue up securely. When "Verified", the backend recalculates optimal fleet routes on the fly.
*   **SHG Dashboard**: Allows Self-Help Group supervisors to track assigned bins securely and look at their historical impact/compensation metrics.
*   **Recycler Portal**: Designed for scrap buyers to track incoming material shipments and bid histories.

## 5. MOBILE APP - COMPLETED FEATURES
Built in React Native (Expo) across 3 distinct navigator flows.
*   **Citizen/Public App**: No login required. Geofenced camera access to snap photos of overflowing bins. Citizens can track the resulting report validation and simulated Live Truck ETA.
*   **Collector App**: Login secured. Features a Map interface mapping out the specific sequence of stops dictated by the OR-Tools optimizer. They click a button when physically at the bin to mark it complete. Features history and KPI stat screens.
*   **SHG App**: Allows ground workers to perform check-ins on specific bins, update the capacity level (0-100%), and view upcoming collection schedules.

## 6. AI FEATURES
1.  **Gemini AI Computer Vision (`image_analyzer.py`)**
    *   Takes the raw BLOB uploaded by users and pipes it to `gemini-1.5-flash`.
    *   *Payload generation*: Forces a JSON response output estimating: `fill_level (0-100)`, `waste_type (plastic/mixed/organic)`, `urgency`, `observations`, and a `confidence` rating.
2.  **Route Optimization VRP (`route_optimizer.py`)**
    *   Powered by Google OR-Tools. Analyzes bins exceeding the threshold (e.g., >60% fill level) or flagged urgently by citizens.
    *   Builds a Haversine distance matrix routing nodes based on minimizing total driving distance while respecting truck weight capacity constraints and strictly prioritizing "urgent" nodes using custom penalty multipliers.

## 7. GEOFENCING
Geofencing strictly prevents abuse. 
*   **Implementation**: Dual-validated. Frontend calculates proximity logic (`mobile/src/utils/geofence.js` using Haversine equations). Backend redundantly checks coordinates before transacting database modifications.
*   **Where it applies**:
    1.  *Public Users* cannot issue an overflowing bin report unless their phone GPS proves they are within 50 meters of the registered bin node.
    2.  *Collectors* cannot mark a bin as "Collected" on their route unless their truck location API ping is within 50m of the bin.

## 10. WHAT IS PENDING / NOT IMPLEMENTED
*   **Live Collector Tracking**: The exact GPS tracking socket between the Collector mobile app and the Citizen app is currently utilizing simulated coordinate offset logic (`backend/routers/public.py`).
*   **Recycler Auth**: "Recycler" represents an abstract entity linked to market sales, but lacks a dedicated `UserRole` login sequence mirroring the others natively in the App auth flow.
*   **Push Notifications**: Need native Expo Push Notifications for alerting collectors when new routes are calculated or dispatched in real time.
*   **File Storage**: Currently, photos are saved onto the local backend filesystem inside `/uploads/`. A cloud bucket provider snippet (AWS S3/GCP) integration would be optimal for scale.
